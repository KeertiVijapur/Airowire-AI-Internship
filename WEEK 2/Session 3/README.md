# Tools in Data Science (TDS) – API Development & Testing Workflows

This document summarizes the third session of Week 2 of the Tools in Data Science (TDS) course, which focused on practical API usage, interacting with services programmatically, and understanding real-world development workflows.

---

## Session Details
**Course:** Tools in Data Science (TDS)

**Mode:** Recorded Online Session

## Session Overview
The session expanded on earlier deployment concepts and demonstrated how applications communicate through APIs. It covered sending requests to services, understanding request/response behavior, testing APIs, and integrating APIs into development workflows. The session emphasized practical understanding of how backend services interact with applications and tools.

---

## Topics Covered

### API Communication Fundamentals
Understanding how client applications communicate with servers using HTTP requests and responses.

---

### Working with API Endpoints
Sending requests to endpoints and analyzing returned data.

---

### GET and POST Requests in Practice
Hands-on usage of HTTP methods and understanding their purposes.

---

### Testing APIs
Using browser tools and command-line utilities to validate API functionality.

---

### Handling Responses and Errors
Understanding status codes, error handling, and debugging API issues.

---

### Integrating APIs into Development Workflow
Using APIs as part of applications and development pipelines.
