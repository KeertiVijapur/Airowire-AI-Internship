# Tools in Data Science (TDS) – WSL Setup & Git Introduction

This document summarizes the second session of the Tools in Data Science (TDS) course, which focused on setting up a Linux-based development environment using Windows Subsystem for Linux (WSL) and introducing Git and GitHub workflows for version control and project documentation.

---

## Session Details
**Course:** Tools in Data Science (TDS)  
**Session:** Week 1 – Session 2  
**Mode:** Recorded Online Session  
**Focus Area:** WSL installation, environment setup, and Git workflow  

---

## Session Overview
The session provided a hands-on walkthrough of installing and configuring Windows Subsystem for Linux (WSL) and Ubuntu. It also introduced version control concepts using Git and demonstrated how GitHub is used for managing repositories and tracking project progress. Troubleshooting methods and best practices for development environments were also discussed.

### Course Portal and Session Overview
Understanding course navigation, session flow, and assignment expectations.

---

### Understanding Terminals and WSL
Introduction to command-line interfaces and their importance in Data Science and AI workflows.  
Explanation of how WSL allows Linux environments to run inside Windows systems.

---

### WSL vs VirtualBox
Comparison between WSL and VirtualBox environments.

#### WSL Advantages
- Lightweight and faster  
- Better integration with Windows  
- Easier setup and resource management  

#### VirtualBox Advantages
- Full virtual machine capabilities  
- Useful for complete OS isolation
