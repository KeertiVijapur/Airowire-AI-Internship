## Data Visualization (https://www.youtube.com/watch?v=XkxRDql00UU)

Data visualization is a crucial step in the data analysis pipeline. After cleaning, modeling, and analyzing data, insights must be communicated effectively. This involves two key aspects:

* **How you show the data** (visual design)
* **How you tell the story** (data storytelling)

This module focuses on the **“show” part** — designing outputs using visualization tools.

---

## Importance of Data Visualization

* Helps convert complex data into **clear and understandable insights**
* Enables **better decision-making**
* Bridges the gap between **analysis and communication**
* Makes data accessible to both **technical and non-technical audiences**

---

## Types of Visualization Tools

### 1. General-Purpose Tools

These tools are widely used and cover most visualization needs:

* **Excel**

  * Create charts, dashboards, and even advanced/custom visualizations
  * Useful for quick analysis and reporting

* **Business Intelligence (BI) Tools**

  * **Power BI**
  * **Tableau**
  * **Google Data Studio**

  These tools help in:

  * Building interactive dashboards
  * Handling large datasets
  * Creating real-time reports

---

### 2. Specialized Tools

These tools are designed for specific visualization purposes:

* **Flourish**

  * Used for animated visualizations
  * Examples: bar chart race, motion charts

* **Kumu**

  * Used for **network visualization**
  * Ideal for representing relationships (e.g., social networks, actor connections)

* **QGIS**

  * Used for **geospatial visualization**
  * Helps create maps and location-based insights

* **RAWGraphs**

  * Useful for creating unique and complex visualizations

---

## Key Concepts Learned

### Dashboards

* One of the most common ways to present data
* Combine multiple charts and KPIs in one view
* Not always the best, but widely used

### Visualization Design

* Focus on clarity and simplicity
* Avoid clutter and unnecessary complexity
* Choose the right chart type for the data

### Tool Selection Strategy

* Use tools you are **most familiar with**
* Ensure the tool can **effectively solve the problem**
* Different tools are suited for different tasks
