### Making Open Data Useful (https://www.youtube.com/watch?v=7NCUE02l1DE)

**Core Idea:**
Open data is often *available* but not *usable*. Your job is to make it usable.

---

### 1️ Work in 3 Steps

**Fetch → Parse → Aggregate**

* **Fetch:** Download raw data (use Network tab + HAR). Save originals.
* **Parse:** Clean HTML/PDF/Excel → structured JSON.
* **Aggregate:** Combine into one clean CSV/SQLite file.

---

### 2️ Use LLMs for Structure, Not Insight

Good for:

* Classification
* Extracting structured JSON
* Normalizing messy text

Not for:

* “Finding what’s interesting”

---

### 3️ Make Everything Linkable

If it can’t be linked → it doesn’t exist.
Use permalinks for searches, filters, and pages.

---

### 4️ Prefer Static, Browser-Based Apps

* Low cost
* No maintenance
* Runs forever

---

### 5️ Documentation is Essential

Include:

* Data dictionary
* Example queries
* Reproducible notebook

Makes journalists and others actually use your data.

---

### 6️ Design for Real Users

Avoid complex dashboards.
Use simple search boxes and intuitive UI.
